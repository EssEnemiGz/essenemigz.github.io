Github Pages - Rutas

En este artículo se enseñará a usar rutas en Github Pages y cómo aprovechar su funcionamiento siendo desarrollador.

Para ponernos en contexto, una ruta sería lo siguiente "essenemigz.github.io/ruta", en este blog se puede apreciar una, la cual es de una carpeta seguida de un archivo; esto se puede extender a repositorios que tengan Github Pages habilitado.

Esto nos puede ser útil para tener una mayor organización en nuestras páginas, o para acceder a contenido estático que ya hayamos alojado en otro repositorio.

Requisitos previos:

  • Tener un repositorio con Github Pages habilitado.
  • Saber lo básico de Github Pages.
  • Saber lo básico del arbol de directorio de Unix(opcional).

Para empezar, esto de las rutas se puede hacer de tres formas distintas:

  1. A base de carpetas y archivos:

    Esto significa que las rutas se van a ver así "user.github.io/carpeta/archivo.html"

  2. A base de archivos:

    Lo que significa que solo vamos a tener un directorio en nuestro repositorio, donde van a estar todos los archivos; las rutas se ven así "user.github.io"(para el index.html) "user.github.io/archivo2.html" y así con los demás

  3. Conectando con otros repositorios:

    Así es, puedes conectar tus repositorios a tu página web si tienes contenido estático alojado ahí, esto es muy útil, haciendo que podamos dividir las páginas a distintos repositorios, haciendo que sean más simples de administrar. El funcionamiento de las rutas sería igual al de las carpetas y archivos, se verían así "user.github.io/repos/carpeta/archivo.html" o "user.github.io/repos/archivo.html"

Tenga esto en cuenta cuando aloje su página web; Github Pages muestra la ruta de esta forma cuando hay un archivo "index.html" en la raíz del repositorio: "user.github.io"

Supongamos que tenemos un repositorio con tres archivos(articles.html, index.html, ports.html) cuando entremos a nuestra web, el archivo que se nos va a mostrar es el index.html. Para enlazar la web con los demás tenemos que agregar el nombre de los otros archivos al dominio. Es decir "user.github.io/articles.html", de esta forma se nos va a mostrar el archivos "articles.html".


Funcionamiento de la ruta "/" en los archivos

Supongamos que queremos usar una imagen que está en el directorio "..", la forma común de acceder a esta es especificando en el documento que esta está en el directo "..", pero esto puede hacer nuestra plantilla poco entendible si la imagen está muy lejos; la solución a esto es aprovecharnos de la ruta "/". La ruta "/" funciona para especificar que estamos en el home de nuestra web, es decir, si la imagen está en /articles y nosotros estamos en /tutoriales/guia podemos acceder a la imagen con la ruta "/articles/img.png". Así funciona la ruta "/" en nuestra web.
